# 현재 상태 백업 프롬프트

현재 프로젝트 상태를 백업해 주세요.

## 작업
1. 현재 Git 상태를 확인한다.
2. 수정된 파일 목록을 확인한다.
3. 백업 방법을 제안한다.
4. 사용자가 승인하면 백업 브랜치 또는 백업 커밋을 만든다.
5. 백업 후 복구 방법을 함께 기록한다.

## 권장 백업 방식
- Git이 초기화되어 있으면 `backup/YYYYMMDD-HHMM` 브랜치를 생성한다.
- Git이 없으면 프로젝트 루트 밖에 압축 백업 폴더를 만든다.

## 중요
- 사용자 승인 없이 Git push를 하지 않는다.
- 사용자 승인 없이 파일을 삭제하지 않는다.
- 백업 전 변경 파일 목록을 반드시 보여준다.

## 출력
# 현재 Git 상태
# 백업 방식
# 백업 이름
# 백업 대상
# 복구 방법
