# 백업에서 복구 프롬프트

이전 백업 상태로 복구해 주세요.

## 작업
1. 사용 가능한 백업 목록을 확인한다.
2. 현재 작업 상태와 충돌 가능성을 확인한다.
3. 어떤 백업으로 복구할지 사용자에게 확인한다.
4. 승인 후 복구를 수행한다.
5. 복구 후 파일 상태와 실행 가능 여부를 확인한다.

## 중요
- 사용자 승인 없이 복구하지 않는다.
- 현재 변경 사항이 사라질 수 있으면 반드시 경고한다.
- `git reset`, `git checkout`, `git clean` 같은 명령은 실행 전 반드시 설명하고 승인받는다.
- 복구 후 변경된 파일 목록을 보고한다.

## 출력
# 사용 가능한 백업
# 복구 대상
# 위험 경고
# 실행한 복구 명령
# 복구 결과
