## 5. PRD 핵심 내용 요약

### 5.1 프로젝트 정의

- **Epic:** 불변식 기반 사고 훈련 시스템 구축
- **목표:** 알고리즘 난이도보다 TDD·계약·불변식·Dual-Track 훈련
- **범위:** UI/DB/Web 없이 콘솔·pytest 기반 순수 로직

### 5.2 고정 I/O 계약

- **Input:** 4×4 `int[][]`, `0`=빈칸 2개, 값 `0` 또는 `1~16`, 0 제외 중복 금지
- **Output:** `int[6]` `[r1,c1,n1,r2,c2,n2]`, 좌표 1-index
- **Solver:** small-first → 실패 시 reverse → 모두 실패 시 `DOMAIN-ERR-NO_VALID_PLACEMENT`

### 5.3 Functional Requirements

| FR | 기능 | Layer |
|---|---|---|
| FR-01 | Input Verification | Boundary |
| FR-02 | Blank Coordinate Discovery | Domain |
| FR-03 | Missing Number Discovery | Domain |
| FR-04 | Magic Square Validation | Domain |
| FR-05 | Two-Combination Solver + Result Formatting | Domain + Boundary |

### 5.4 Dual-Track TDD

- **Track A:** Boundary 입력 검증, 출력 schema, Domain 미호출, 오류 응답
- **Track B:** BlankFinder, MissingNumberFinder, Validator, Solver 불변식

### 5.5 PRD에서 확정한 정책

- 입력 검증 실패 → Domain resolver **0회 호출**
- 두 조합 모두 실패 → **구조화 실패 응답** `DOMAIN-ERR-NO_VALID_PLACEMENT`
- 입력 행렬 처리 전후 **deep equality 유지**
- Data Layer(Report/02 §3) → **Out-of-Scope**
