# 4x4 MagicSquare Cursor Rules MDC 전환 보고서

## 1) 작업 개요
- 기존 단일 규칙 파일인 `.cursorrules`를 유지한 상태에서,
  Cursor AI가 인식하는 `.cursor/rules/*.mdc` 구조를 신규 구성함
- 규칙 적용 방식을 혼합형으로 설정함:
  - 공통 규칙: `alwaysApply: true`
  - 문맥 규칙: `globs` 기반 `alwaysApply: false`

## 2) 생성 파일
- `.cursor/rules/magicsquare-project.mdc`
- `.cursor/rules/magicsquare-python-code-style.mdc`
- `.cursor/rules/magicsquare-ecb-architecture.mdc`
- `.cursor/rules/magicsquare-tdd-testing.mdc`
- `.cursor/rules/magicsquare-forbidden.mdc`

## 3) 규칙 분리 전략
### 공통 규칙
- `magicsquare-project.mdc`
  - 프로젝트 목적
  - 기본 레포 구조
  - 작업 전/중/후 기본 행동 지침
  - `alwaysApply: true`

### Python 코드 규칙
- `magicsquare-python-code-style.mdc`
  - PEP8, 타입힌트, Google docstring 기준
  - `globs: "**/*.py"`

### 아키텍처 규칙
- `magicsquare-ecb-architecture.mdc`
  - ECB 레이어 책임 정의
  - 의존 방향(`boundary -> control -> entity`) 제한
  - `globs: "src/**/*.py"`

### TDD/테스트 규칙
- `magicsquare-tdd-testing.mdc`
  - Red/Green/Refactor 단계 수칙
  - pytest + AAA 패턴 지침
  - `globs: "**/*.py"`

### 금지 패턴 규칙
- `magicsquare-forbidden.mdc`
  - `print()` 디버깅, bare `except:`, 매직넘버/문자열, 테스트 약화 금지
  - 대체 권장 패턴 명시
  - `globs: "**/*.py"`

## 4) 확인 사항
- `.cursor/rules` 폴더 하위에 `.mdc` 파일 5종이 생성되었음
- IDE 린트 진단 기준 오류 없음
- 기존 `.cursorrules`는 보존되어 있어 비교/참조 가능함

## 5) 후속 권장 작업
- 중복 관리 관점에서 `.cursorrules`를 최종 비활성화하거나 보관 전용으로 전환
- 신규 규칙은 기존 파일 확장보다 주제별 `.mdc` 추가를 우선 적용
