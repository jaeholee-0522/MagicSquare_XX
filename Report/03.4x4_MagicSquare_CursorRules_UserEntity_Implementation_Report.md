# 4x4 MagicSquare Cursor Rules + User Entity 구현 보고서

## 1) 작업 개요
- `.cursorrules` 파일을 생성하고 MagicSquare 프로젝트 기준 규칙을 상세화함
- `tdd_rules`, `forbidden`, `ai_behavior`를 포함한 전체 섹션을 확장함
- ECB 패턴 기준 `User` 엔티티 및 pytest 테스트(AAA 패턴)를 신규 작성함

## 2) 생성/수정 파일
- `.cursorrules`
- `src/__init__.py`
- `src/entity/__init__.py`
- `src/entity/user.py`
- `tests/entity/test_user.py`

## 3) .cursorrules 반영 사항
### code_style
- Python 3.10+
- PEP8 엄격 준수
- 타입힌트 필수
- Google 스타일 docstring
- max line length 88

### architecture (ECB)
- boundary: 외부 입출력
- control: 비즈니스 로직
- entity: 도메인 규칙/데이터
- 의존성 방향:
  - boundary -> control
  - control -> entity
  - entity -> (boundary/control) 금지

### tdd_rules
- red/green/refactor를 `description`, `rules`, `must_not` 구조로 세분화
- RED 없이 구현 금지, GREEN에서 리팩터링 금지, REFACTOR에서 동작 변경 금지 반영

### testing
- framework: pytest
- pattern: AAA
- coverage_minimum: 80%
- fixture_scope 규칙 및 test_ 접두사 명명 규칙 반영

### forbidden
- `print()`, 하드코딩 상수, `except:`(bare except) 금지
- 각 항목에 pattern/reason/alternative 추가

### file_structure
- ECB 폴더 구조 주석 트리 명시 (`src/boundary`, `src/control`, `src/entity`, `tests/*`)

### ai_behavior
- 코드 전/중/후로 나누어 준수 규칙 명시
- 테스트 선확인, ECB 경계 위반 금지, 타입힌트 없는 함수 금지, TDD 위반 경고 포함

## 4) User 엔티티 구현 요약
### 위치
- `src/entity/user.py`

### 주요 내용
- `@dataclass(frozen=True, slots=True)` 기반 불변 엔티티
- 필드:
  - `user_id: int`
  - `name: str`
  - `email: str`
- 검증:
  - `user_id > 0`
  - `name` 공백 금지
  - `email` 기본 포맷 검증
- 도메인 메서드:
  - `rename(new_name: str) -> User`
  - `change_email(new_email: str) -> User`

## 5) 테스트 구현 요약
### 위치
- `tests/entity/test_user.py`

### 케이스
- 유효한 값으로 생성 성공
- 공백 이름 입력 시 `ValueError`
- 잘못된 이메일 형식 입력 시 `ValueError`
- `rename()` 호출 시 새 인스턴스 반환
- `change_email()` 호출 시 새 인스턴스 반환

### 스타일
- pytest 사용
- AAA 패턴(Arrange / Act / Assert) 준수
- 함수명 `test_` 접두사 준수

## 6) 검증 결과
- IDE 린트: 오류 없음
- 테스트 런타임 실행: 현재 환경 Python 런타임 미설치로 미실행
  - `python -m pytest -q` 실패 (런타임 없음)
  - `py -3 -m pytest -q` 실패 (적합한 Python runtime 없음)

## 7) 결론
