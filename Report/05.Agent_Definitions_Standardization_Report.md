# 작업 보고서

## 작업 개요
- 목적: `.cursor/agents`와 `.cursor/prompts` 구성을 요청사항(A-1~A-9)에 맞게 정비하고, 에이전트 정의 형식을 통일함.
- 범위: 에이전트 신규 생성/이름 변경/내용 보강/출력 형식 정렬, 멀티 에이전트 실행 프롬프트 파일 추가.

## 주요 변경 내용
1. 에이전트 정의 통일
- 공통 섹션 스타일(`역할`, `주요 책임`, `작업 방식`, `금지`, `출력 형식`)로 정리.
- `code-reviewer`, `security-reviewer`, `test-reviewer` 포함 다수 에이전트 출력 형식 헤더 기반으로 통일.

2. 이름/역할 정합성 개선
- `optimization-engineer` -> `system-optimization-engineer`
- `ux-designer` -> `ux-design-advisor`
- `product-manager` -> `product-planning-manager`
- `qa-engineer` -> `quality-assurance-engineer`
- `backup-agent` -> `backup-report-github-manager`

3. 누락 에이전트/프롬프트 추가
- `frontend-developer`, `code-bug-analyzer`, `performance-optimizer` 추가.
- 멀티 에이전트 협업 및 실행/백업/복구 프롬프트 4종 추가.

## 실행한 명령
- `git status --short`
- `ls`
- `mkdir "Prompt"`
- 파일 생성/수정/삭제는 에디터 패치 도구로 수행

## 테스트 결과
- 문서성 변경 중심으로 실행 테스트는 생략.
- 변경 파일 대상 lint 점검에서 오류 없음 확인.

## 남은 이슈
- `git status --short` 기준으로 기존 워킹트리에 다수 미추적/변경 파일이 존재함.
- 프롬프트 백업 저장 위치는 `Prompt/`로 통일 유지 필요.

## 다음 작업 제안
- 백업/리포트 파일명 규칙을 `Report`와 `Prompt`에서 단일 컨벤션(점/언더스코어)으로 확정.
- 필요 시 이번 변경분만 별도 커밋 단위로 스테이징하여 히스토리 가독성 확보.
