### 5.4 TDD 흐름 고정 원칙

```
Scenario → Acceptance Criteria → RED Test ID → Test Skeleton
→ Run Test → Confirm Failure(RED) → GREEN Task Candidate → REFACTOR Candidate
```

- **RED:** 테스트 실패 상태 확인 (미구현으로 인한 기대 실패)
- **GREEN Task Candidate:** RED 실패를 통과시키기 위한 **최소 구현 작업 후보**만 기술
- **REFACTOR Candidate:** GREEN 이후 **구조 개선 후보**만 기술 (실제 수행 없음)
