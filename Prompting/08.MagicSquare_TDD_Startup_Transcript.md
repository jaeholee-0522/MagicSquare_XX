# Magic Square 4x4 — TDD 시작 준비 (To-Do List · README) Prompting Transcript Export

아래는 TDD 시작 준비 세션의 사용자-어시스턴트 대화를 순서대로 정리한 export transcript이다.

---

## 세션 범위

- Turn 1: PRD 기반 개발 To-Do List 작성 (Scenario → AC → RED Tracking Board)
- Turn 2: README.md 초안 작성 (TDD 시작 선언·개발 가이드)
- Turn 3: Report·Prompting Transcript Export

---

## Turn 1 — 개발 To-Do List 작성

### User

Senior Software Architect이자 Dual-Track UI + Logic TDD 코치 역할로, PRD 기반 **개발 To-Do List** 작성을 요청함.

**제약**

- 구현 코드·테스트 코드·파일 생성/수정 **금지**
- To-Do List Markdown 본문만 출력
- RED = “실패 상태 확인”으로 표현
- GREEN = “RED 실패를 통과시키기 위한 최소 구현 작업 후보”
- REFACTOR = “GREEN 이후 구조 개선 후보” (실제 수행 없음)

**출력 구조**

1. Epic
2. User Stories
3. Scenario → AC → RED → GREEN Tracking Board (표)
4. RED Start Checklist
5. Traceability Summary (표)

**필수 Scenario**

- None 입력, 4×4 아님, 빈칸 개수/값 범위/중복 오류
- 빈칸 row-major, 누락 숫자 오름차순
- 행/열/대각선 합 34, small-first/reverse/both-fail
- int[6], 1-index 좌표

**참고 문서**

- `docs/PRD_MagicSquare.md`
- `Report/02`, `Report/03`, `Report/04`, `Report/06`
- `.cursorrules`, `.cursor/rules/*.mdc`

### Assistant

참고 문서(PRD, Report/02~04, Report/06, `.cursor/rules/*.mdc`)를 읽고 **MagicSquare 4x4 개발 To-Do List**를 Markdown으로 출력함.

**주요 산출**

| 구분 | 내용 |
|---|---|
| Epic | Epic-001: 불변식 기반 사고 훈련 시스템 구축 |
| User Story | US-001~005 (입력 검증, 빈칸, 누락 숫자, 마방진 검증, Solver+포맷) |
| Tracking Board | TASK-001~020, 20행 |
| RED Test ID | `RED-BND-001`~`007`, `RED-DOM-001`~`010`, `RED-INT-001`~`003` |
| Track | Boundary RED / Logic RED / Integration RED 분리 |
| ECB Layer | Entity / Control / Boundary 명시 |
| Checklist | RED Start 11항목 |
| Traceability | 16 Concept/Invariant 행 |

**Tracking Board 핵심 Task**

- TASK-001~005: Boundary 입력 검증 (None, SIZE, ZERO, RANGE, DUPLICATE)
- TASK-006~007: 출력 계약 (길이 6, 1-index)
- TASK-008~017: Domain (BlankFinder, MissingNumberFinder, Validator, Solver)
- TASK-018~020: Integration (Domain 미호출, end-to-end, public API)

**원칙 준수**

- Expected RED Failure: “미구현 — **실패 상태 확인**”
- GREEN Task Candidate: 최소 구현 후보만
- REFACTOR Candidate: 구조 개선 후보만

## Turn 2 — README.md 초안 작성

### User

Technical Writer이자 Dual-Track TDD 코치 역할로, **README.md 초안** 작성을 요청함.

**제약**

- 구현·테스트 코드·파일 생성/수정 **금지**
- README Markdown 본문만 출력
- RED = “테스트 실패 상태를 확인하는 단계”
- GREEN / REFACTOR = Turn 1과 동일 (후보만, 실제 수행 없음)

**출력 구조 (10개 섹션)**

1. Project Start Declaration
2. PRD Summary
3. TDD Development Flow
4. Development Methodology
5. ECB Role Separation (표)
6. Scenario → AC → RED Tracking Board (표, 필수 Scenario 전건)
7. RED Start Checklist
8. Quality Gates
9. Reference Documents
10. Current Project Status

### Assistant

`docs/PRD_MagicSquare.md`, Report/02~06, `.cursor/rules/*.mdc` 등을 참고하여 **MagicSquare 4x4 TDD Practice** README 본문을 Markdown으로 출력함.

**주요 산출**

| § | 핵심 |
|---|---|
| §1 | PRD 기반 TDD 시작 준비 선언, RED 정의, 구현 전 가이드 |
| §2 | 목적, 학습 목표, 도메인 규칙, I/O 계약, ESC-01~07 |
| §3 | 8단계 TDD 흐름 (Scenario → … → REFACTOR) |
| §4 | Dual-Track, ECB, Traceability, RED→GREEN→REFACTOR |
| §5 | Entity / Control / Boundary 표 |
| §6 | 17행 Tracking Board (필수 Scenario 15건 + 통합 관련) |
| §7 | RED Start Checklist 11항목 |
| §8 | Domain ≥95%, Boundary ≥85%, pytest, AAA, forbidden |
| §9 | Reference Documents + 실제 경로 매핑 + 문서 간 관계도 |
| §10 | 현재 미구현 상태, 권장 RED 시작 순서 (Boundary Track A 우선) |

**Turn 1 대비 차이**

- README는 Task ID 없이 Scenario 중심 17행
- §9 Reference Documents + 문서 간 관계 ASCII 다이어그램 추가
- `pyproject.toml` 미작성 상태 명시
